Week 1:
    1. IDD/IDE = Integrated development environment
    2. DOM = document object model
    3. HTML = Hyper-text markup language
    4. CSS = Cascading style sheets
    5. Terminal/Command line = A text-based interface to the system
    6. Shell = The part of your operating system that defines how the terminal will behave/look
    7. Bash = Bourne Again SHell


Week 2:
    1. DRY = Don't repeat yourself
    2. href = Hyper-text reference
    3. User story = the scenario that the hypothetical "user" wants to see on the page -
        you have to build it out. Can be detailed or vague.
    4. Alt text = text that displays when the browser can't load the element or for impared users.
        This text is also a great way to get found in search results.
    5. HTML containers = The commonly used sections that compose your HTML file.
        Containers help with readability and organization
    6. Attributes = the extra information inside an HTML element that povides additional data.
    7. CSS selectors: Element/class/ID = the selector determining which bits of the HTML page
        will be styled. Element seletcs all of that particular type of element, Class (.) selects all elements
        with the that class attribute, and ID (#) seletcs only the particular element with that ID listed.

    2. SDET = Software Design Engineer in Test
    3. SDE = Software Design Engineer
    4. Program = A list of computer instructions that tell a computer what to do.
    5. (GIT) ACP = add commit push
    6. camelCasing = the standard style of capitalization in JavaScript, where theLaterWordsAreInCaps.
        because it looks like camelHumps.

Week 4:
    1. js argument = the input, eg the 9 in multiply(9)
    2. js parameter = variables in a function name that recieve an input
    3. js function declaration = a function that is bound to an identifier or name
    4. js modulous operator = %, finds the remainder
    5. js operators = the symbols used to assign values, compare values, perform arithmetic operations, etc.
    6. lexical order = the numerical value assigned to strings is based off of the first letter. This is the mathematical term for alphabetical order. 
    7. Boolean = true and false values


Week 5:
    1. scope = the degree to which an object is accessible by other functions. Eg. an object within a function is local to that function, where as an object outside of all functions has global scope.
    2. loops = while loops and for loops let you perform an action multiple times, or only in certain cases.
    3. i = itteration
    4. API = application programing interface
    5. DBMS = data base management system
    6. Front end = “client-side” development, where the focus is on what users see
    7. Open-source = software that is available for public use, change, and improvement



Week 6:
    1. "this" = specifies the object or variable within that particular scope. changes when called in a different location.
    2. constructor function = a function that creates a new object
    3. global = accessable from any location
    4. local = accessable only within the object
    5. event listener = "listens" for a specific event to happen on the web page, which calls the action assigned
    6. higher order function = functions within functions. some examples are the functions you can add within .filter .map and .reduce
    7. arity = the number of expected arguments in a function. Eg. function (a,b,c){} has an arity of 3.
